# Nathan's Game Design Suggestions

Feedback from Nathan (Valokor) after playtesting over lunch. These are design proposals for the next iteration — to be triaged into core-priorities, nice-to-haves, or backlog.

## Scrap Generation

- [ ] Rate-limit player clicks on scrap piles to 5–10/sec
- [ ] Capturing additional scrap piles should auto-generate 1 scrap/sec each

## ATK / DEF Rework

- [ ] Attack Bots: click an opponent's tile to assault it over several seconds (~5s)
- [ ] Neutral tiles should also require time to capture (~1s), requiring at least 1 Attack Bot
- [ ] Progress bar / graphic when capturing a tile (show attack bot on tile)
- [ ] Number of Attack Bots limits how many tiles you can assault simultaneously
- [ ] Defense Bots: select your own tiles to make them immune to conversion
- [ ] Number of Defense Bots limits how many tiles can be protected at once
- [ ] Visual indicator on defended tiles

## Additional Scrap Upgrades

- [ ] Improved Defenses upgrade: increase time opponents need to capture your tiles
- [ ] Improved Attackers upgrade: decrease time to capture tiles
- [ ] Improved Scrap Collection upgrade: +1 auto scrap/sec
- [ ] All upgrades cost 10% more than the previous level
